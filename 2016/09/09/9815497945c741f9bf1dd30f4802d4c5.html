clone vs dup

Sep 9, 2016

clone和dup都是浅复制

clone会连同单例类所拥有的都复制，而dup则不会

o = Object.new
def o.foo
  42
end

o.dup.foo   # raises NoMethodError
o.clone.foo # returns 42

class << o
  A = 5
end

o.dup.singleton_class::A   # NameError: uninitialized constant A
o.clone.singleton_class::A # return 5


clone会复制frozen状态，而dup不会

class Foo
  attr_accessor :bar
end
o = Foo.new
o.freeze

o.dup.bar = 10   # succeeds
o.clone.bar = 10 # raises RuntimeError


用ruby伪码来表示的话，是这样：

class Object
  def clone
    clone = self.class.allocate

    clone.copy_instance_variables(self)
    clone.copy_singleton_class(self)

    clone.initialize_clone(self)
    clone.freeze if frozen?

    clone
  end

  def dup
    dup = self.class.allocate
    dup.copy_instance_variables(self)
    dup.initialize_dup(self)
    dup
  end

  def initialize_clone(other)
    initialize_copy(other)
  end

  def initialize_dup(other)
    initialize_copy(other)
  end

  def initialize_copy(other)
    # some internal stuff (don't worry)
  end
end


因此，如果需要在复制时进行一些拦截工作，可重写initialize_clone/initialize_dup（注意调用super，使核心的initialize_copy得以运作）

例如：

class A
  attr_reader :proto
  def initialize_clone origin
    @proto = origin
    super
  end
end

a = A.new

b = a.clone

a == b.proto # true


注意如果是类复制，则def self.initialize_clone，且设置实例变量要用instance_variable_set，而不能直接@xxx = xxx
